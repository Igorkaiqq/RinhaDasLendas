# RinhaDasLendas Design System

## Visão

O RinhaDasLendas é uma plataforma para gerenciamento de partidas internas de League of Legends.

A identidade visual deve transmitir:

* Competitividade
* Modernidade
* Organização
* Comunidade
* Tecnologia

O sistema deve parecer um produto SaaS moderno e não uma aplicação gamer exagerada.

Referências:

* Linear
* Discord
* Valorant Tracker
* OP.GG
* Notion

## Direção Visual Atual

A aplicação deve combinar clareza de SaaS moderno com energia competitiva de eSports.

O visual aprovado para evolução usa:

* base escura profunda;
* superfícies em camadas;
* acentos roxo e azul para ação e tecnologia;
* verde, amarelo e vermelho apenas para estados semânticos;
* detalhes sutis de arena competitiva, sem exagero visual;
* tipografia limpa, com uso pontual de fonte mono para dados, rotas, status e elementos de sistema.

A interface não deve parecer ERP, CRM ou dashboard administrativo genérico.

A assinatura visual do produto é uma "arena de draft": superfícies escuras, acentos energéticos controlados, estados claros e foco em jogadores, times e escolhas.

---

# Filosofia Visual

## Dark First

Todo o sistema é projetado inicialmente para Dark Mode.

Light Mode poderá existir futuramente.

---

## Product First

A informação é o elemento principal.

A interface deve destacar:

* jogadores;
* partidas;
* estatísticas;
* drafts;
* séries.

Evitar excesso de ilustrações.

---

## Cards Como Elemento Principal

Praticamente toda informação deve ser apresentada em cards.

Exemplos:

* Jogador
* Time
* Partida
* Série
* Draft
* Estatísticas

---

## Poucas Cores de Destaque

O sistema utiliza:

* Roxo (principal)
* Azul (secundário)
* Verde (sucesso)
* Vermelho (erro)

Nada além disso.

Variações translúcidas dessas cores podem ser usadas para hover, focus, badges e estados, desde que derivadas dos tokens oficiais.

---

## Hierarquia Visual

1. Dados
2. Ações
3. Navegação
4. Elementos decorativos

Nunca o contrário.

---

# Componentes Principais

## Cards

Elemento dominante da aplicação.

## Tabelas

Usadas para:

* rankings
* histórico
* estatísticas

## Badges

Utilizadas para:

* elo
* posição
* status

## Drawer

Preferido para edição rápida.

## Modal

Utilizado apenas para confirmação.

---

# Navegação

Menu lateral fixo.

Itens previstos:

* Dashboard
* Jogadores
* Filas
* Drafts
* Partidas
* Séries
* Estatísticas
* Configurações

---

# Responsividade

Desktop primeiro.

Breakpoints:

* 1440+
* 1280
* 1024
* 768
* 480

---

# Experiência

A aplicação deve ser rápida.

O usuário deve conseguir:

* encontrar jogadores rapidamente;
* montar times rapidamente;
* registrar partidas rapidamente.

Menos cliques é melhor.

---

# Regra Final

Se existir dúvida entre algo bonito e algo funcional:

Escolher algo funcional.
